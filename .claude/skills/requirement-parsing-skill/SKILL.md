# 需求解析技能

## 技能概述

将用户的自然语言需求转换为结构化的需求文档。为 RTL 生成提供清晰的规格说明。

## 触发条件

- 用户描述硬件模块需求
- 用户要求"生成"、"创建"、"实现"硬件模块
- 用户输入包含功能描述但无明确接口定义

## 执行步骤

### 步骤1：信息提取

从用户描述中提取：

1. **模块名称** - 从需求中识别或根据功能命名
2. **功能列表** - 支持的操作和特性
3. **接口信号** - 输入/输出端口
4. **数据位宽** - 默认 32 位
5. **控制信号** - 使能、复位、操作码等

### 步骤2：需求澄清

如以下信息不明确，主动询问用户：

| 信息 | 默认值 | 询问方式 |
|------|--------|----------|
| 数据位宽 | 32 位 | "数据宽度是 32 位吗？" |
| 接口信号 | IO bundle | "需要哪些输入输出信号？" |
| 操作类型 | 基础运算 | "需要支持哪些运算？" |
| 时序要求 | 组合逻辑 | "需要寄存器输出吗？" |

### 步骤3：生成需求文档

输出到 `docs/requirements_<模块名>.md`：

```markdown
# <模块名> 需求规格

## 1. 模块概述
- 模块名称：
- 功能描述：
- 应用场景：

## 2. 接口定义
| 信号名 | 方向 | 位宽 | 说明 |
|--------|------|------|------|
| clock  | 输入 | 1    | 时钟信号 |
| reset  | 输入 | 1    | 异步复位 |
| io_a   | 输入 | 32   | 操作数 A |
| ...    | ...  | ...  | ... |

## 3. 功能列表
- [ ] 功能 1：描述
- [ ] 功能 2：描述

## 4. 操作码定义
| opcode | 操作 | 说明 |
|--------|------|------|
| 0      | ADD  | 加法 |
| ...    | ...  | ... |

## 5. 验证要求
- 测试用例数量：
- 覆盖场景：
```

## 输出

- `docs/requirements_<模块名>.md` - 需求规格文档
- 关键信息摘要（用于后续 skill 调用）

## 质量检查

- [ ] 模块名称明确
- [ ] 所有接口有明确定义
- [ ] 功能列表完整
- [ ] 操作码定义清晰
- [ ] 无歧义需求
